  • 第1章 数据类型

    1. 数据类型

      五种基元类型：number、string、boolean、null、undefined

      一种对象类型：包括Function、Array、Date等

    2. 隐式转换

      等于==：类型不同时，尝试类型转换再进行内容判断。

      “1.23” == 1.23、 0 == false、 null == undefined、 new Object() == new Object()、 [1,2] == [1,2]

      严格等于===：先判断类型、再判断内容。类型不同直接返回false

      null === null undefined === undefined *NaN != NaN *new Object() != new Object()
    3. 包装类型

      原始类型number、string、boolean都有对应的包装对象类型。

      当对基本类型进行类似属性操作，比如"abc".length的时候，js会临时创建基本类型的对应包装类型，当该操作完成后，这个临时对象会被立即销毁。

                  > "abc".t = 10;
            < 10
            > "abc".t;
            < undefined
    4. 类型检测

      typeof、instanceof、Object.prototype.toString、constructor(构造器检测)、duck type(特征检测)

      typeof typeof 100 --> "number" typeof true --> "boolean" typeof function --> "function" typeof(undefined) --> "undefined" typeof new Object() --> "object" typeof [1,2] --> "object" *typeof NaN --> "number"。NaN是特殊的number *typeof null --> "object"。历史遗留问题，妥协网站兼容性

      适合基本类型及function检测。遇到null失效。

      instanceof obj instanceof Object

      判断左操作数对象的圆形链上是否有右操作数上的prototype属性。基于此可进行子类对象与父类类型的比较，结果为true。

      适合自定义对象，也可以用来检测原生对象。但不支持跨window、iframe进行判断。坑！

      Object.prototype.toString Object.prototype.toString([]) --> "object Array" Object.prototype.toString(function(){}) --> "object Function" Object.prototype.toString(null) --> "object Null" Object.prototype.toString(undefined) --> "object Undefined" IE6/7/8Object.prototype.toString(null) --> "object Object"

      适合内置对象和基元类型。遇到nullhe undefined失效，返回结果为Object

  • 第2章 表达式和运算符

    1. 表达式 wiki：表达式是指能计算出值得任何可用程序单元 JS权威指南：表达式是一种JS短语，可使JS解释器用来产生一个值。
    2. 运算符

      根据操作数的数量可分为一元、二元、三元运算符。

      根据功能可分为赋值、比较、算术、位、逻辑、字符串、特殊运算符。

      三元运算符。如下例，当c为true时，取值a，否则取值b

      c? a: b var val = true? 1: 2; //val = 1

      逗号运算符。如下例，从前到后依次计算各值，最终取最后值。

      a, b var val = (1, 2, 3); //val = 3

      delete运算符。如下例

      delete obj.x 
                  var obj = {x:1};
            obj.x;  // 1
            delete obj.x;
            obj.x;  // undefined
